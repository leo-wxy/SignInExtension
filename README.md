# SignInExtension

一个面向 Chrome / Arc 的可配置签到扩展。它的目标不是做站点专属脚本，而是通过任务配置、元素拾取和页面状态复检，把常见“每日签到”流程做成可复用的浏览器扩展。

## 当前能力

- 支持多任务签到配置，每个任务绑定目标地址和匹配地址。
- 支持右键菜单快速创建/编辑任务，不必每次先进设置页。
- 支持页面元素拾取，默认优先保存唯一 selector。
- 支持点击前判定“已签到”，避免重复点击。
- 支持点击后复检页面状态，减少“其实成功了但误判”的情况。
- 支持浏览器通知、扩展图标 badge、页面内 toast 三层提示。
- 支持定时自动签到。
- 支持在 Chrome 与 Arc 中加载使用。

## 状态语义

扩展当前把签到结果分成四类：

- `签到成功`：点击后在确认窗口内检测到页面进入已签到状态。
- `已签到`：点击前页面就已经显示为已签到。
- `未确认成功`：按钮已经点击，但确认窗口内没有看到明确成功信号。
- `签到失败`：找不到按钮、按钮不可点击、执行报错等明确失败。

这套状态语义是为了尽量避免“明明页面已经变了，但提示还停在已点击”的误判。

## 任务创建流程

### 方式一：右键快速创建

1. 打开需要签到的页面。
2. 在页面空白处右键。
3. 如果当前地址没有任务，会显示 `任务不存在：创建任务`。
4. 如果当前地址已有任务，会显示 `任务存在：编辑任务`。
5. 页面进入选择模式后，左键点击签到按钮。
6. 在确认面板里确认 selector 并保存。

### 方式二：设置页手动编辑

1. 点击扩展图标。
2. 打开设置页。
3. 在左侧选择任务，右侧直接编辑任务详情。
4. 可以手动调整地址、匹配地址、点击延迟和拾取规则。

## 提示方式

当前提醒分三层：

- 页面内 toast：显示在被执行的签到页面。
- 扩展图标 badge：`成 / 待 / 失`。
- 浏览器通知：全局可见，适合在后台自动签到时提醒。

说明：页面内 toast 不是系统级全局提示；浏览器通知才是更明显的全局提醒。

## 默认任务

仓库里内置了一个默认示例任务：

- 任务名：`牧原签到`
- 地址：`https://muyuan.do/console/personal`
- 地址匹配：`https://muyuan.do/console/*`
- 已签到文案：`今日已签到`、`已签到`

这只是默认示例，实际使用时建议按你的目标站点重新拾取元素并保存任务。

## 本地加载

### Chrome

1. 打开 `chrome://extensions/`
2. 开启“开发者模式”
3. 点击“加载已解压的扩展程序”
4. 选择当前目录

### Arc

Arc 基于 Chromium，加载方式与 Chrome 相同：

1. 打开扩展管理页面
2. 开启开发者模式
3. 加载当前目录

## 常用调试方式

- 在设置页使用单任务 `立即签到`
- 观察页面是否出现已签到状态
- 查看扩展图标 badge
- 查看 popup 中最近一次结果和原因
- 查看浏览器通知是否弹出

如果页面已经变成已签到，但扩展没有识别，优先检查：

- 已签到文案是否稳定
- 点击后页面是否延迟刷新
- selector 是否过短或过于依赖状态 class

## 已知限制

- 默认无法精准判断“需要登录”，只会基于页面状态给出 `未确认成功` 或 `签到失败`。
- 某些站点点击后如果没有稳定的成功信号，仍可能停在 `未确认成功`。
- 当前没有做站点专属接口监听，核心仍然基于 DOM 与页面状态复检。

## 敏感信息检查

当前仓库已做过一轮敏感信息扫描，没有发现硬编码 API Key、私钥、Bearer Token、Webhook 凭证或账号密码。
